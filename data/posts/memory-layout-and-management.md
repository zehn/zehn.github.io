# 内存布局与管理

用户态进程地址空间（从低地址到高地址），标准 64 位 Linux 下，用户态范围通常是 0x00 ~ 0x00007FFFFFFFFFFF（约 128TB）：
| 地址区域 | 存储内容 | 生命周期与分配器 |
|---|---|---|
| 0x00000000 保留区 | NULL 保护区。不可访问，用于捕获空指针非法访问。 | 编译期预留。 |
| Text Segment（代码段）| 机器指令（只读、可执行）。存放 main 函数、库函数的编译后二进制码。|静态分配。程序启动时通过 mmap 加载进内存，全程驻留。 |
| Read-Only Data（.rodata）| 只读全局常量。如字符串字面量 "Hello"、const 修饰的全局变量。|静态分配。放在代码段附近，防止被意外篡改。 |
| Data Segment（.data）| 已初始化的全局变量和静态变量。如 int global = 10; 或 static int count = 5。| 静态分配。程序启动时从可执行文件加载到内存，程序结束释放。 |
| Block Started by Symbol（.bss）| 未初始化的全局变量和静态变量。如 int global_zero。| 静态分配。不占磁盘空间，仅在内存中开辟位置并清零（零初始化）。 |
| Heap（堆）| 动态分配的大/小对象。由 malloc / new / brk 申请。|动态分配。brk 指针向上增长（高地址）。由用户态分配器（ptmalloc/jemalloc）管理，释放后可复用。 |
| Memory Mapping Segment（共享库/匿名映射）| 动态链接库（.so）、mmap 大文件、线程栈。|动态分配。位于堆和栈之间。mmap 系统调用在此区域开辟空间，向下或向上动态浮动。 |
| User Stack（用户栈）| 局部变量、函数调用帧（返回地址、寄存器备份）。| 自动分配。RSP 寄存器指针向下增长（低地址）。大小受 ulimit -s 限制（默认 8MB），按需缺页动态扩容。 |
| 0x00007FFFFFFF 边界 | 内核空间与用户空间分水岭。此处禁止跨权限访问。| — |

内核态地址空间（高地址区域），以 64 位典型布局为例，内核态空间范围为 0xFFFF800000000000 ~ 0xFFFFFFFFFFFFFFFF，所有进程共享同一份内核页表：
| 地址区域 | 存储内容 | 生命周期与分配器 |
|---|---|---|
| Direct Mapping / 线性映射区（PAGE_OFFSET）| 所有物理内存的“永久挂载”。例如物理地址 0 直接映射为 0xFFFF8800_00000000。| 静态/大块管理。由 Buddy System（伙伴系统）管理。kmalloc 从此处分物理连续的小块内存。 |
| VMALLOC 区（动态映射区）| 内核模块（.ko）代码、不连续的物理页拼接成的虚拟连续区（如加载固件）。| 动态分配。通过 vmalloc 申请，页表独立建立，性能略低于直接映射。 |
| Permanent Kernel Mapping（持久内核映射）| 将高端内存（>896MB）的物理页临时映射到内核地址空间。| 用于访问所有物理内存（在 32 位系统上尤为重要）。 |
| Fixmap（固定映射区）| ACPI 表、IOMMU、Local APIC 等硬件寄存器的映射。| 编译期预留固定虚拟地址，运行时动态绑定物理地址。 |
| Kernel Text / Data / BSS（内核自身镜像）| 内核的代码段、全局内核变量（如 current 指向当前进程 task_struct）、内核的 .bss。| 系统启动时由引导程序加载，永不换出。 |
| Kernel Stack（内核栈）| 进程陷入内核态时的调用栈（保存系统调用参数、中断上下文）。| 自动分配。固定大小（通常 8KB~16KB），由 __get_free_pages 分配，不可动态扩容。 |
| Module 区域（内核模块区）| 动态插入的驱动 .ko 文件代码和数据。| vmalloc 或专用模块区域分配，可动态加载/卸载。 |
